01

Posicionamento visual

"vita id parece que o Governo Federal de 2030 contratou a Stripe pra redesenhar a CNH Digital — autoridade brasileira + execução premium internacional."

Não é healthtech jovem (Nubank/Hims). Não é fintech vibrante (Memed/Revolut). Não é gov.br lento (ConecteSUS). Não é hospital empresarial sóbrio puro (Einstein/Sírio).

É a interseção dos quatro: a alma de documento oficial brasileiro + a execução visual da Apple Wallet + a densidade clínica do MyChart + o silêncio premium do One Medical.

02

7 features hero confirmadas

  1. RG da Saúde (cartão) — artefato central, ID do produto
  2. QR Code para médico/socorrista — momento de máximo valor
  3. Exames — organização + entendimento em PT-BR simples
  4. Alergias — destaque vermelho no RG público, life-saving
  5. Medicamentos — lista contínua + cruzamento com alergias
  6. Pré-consulta — paciente chega pronto, médico recebe briefing
  7. Linha do tempo — histórico cronológico unificado

Outras features (premium, ajustes, dados pessoais) ficam secundárias e escondidas.

03

Modelo de navegação

Escolhido: HUB + SHEET VERTICAL (combo Apple Wallet + Carteira Trabalho Digital). Sem tab bar.

Brasileiro lê cartão digital como "documento oficial". Tab bar transformaria o RG num app de delivery. O cartão é o produto, não o menu.

Regra 60-30-10

  • 60% off-white/cream/branco (a marca é o silêncio visual)
  • 30% neutros escuros e cinzas
  • 10% cor de marca + semânticas (acentos, CTAs, status)

Único uso permitido: gradient sutilíssimo verde→teal em hero de landing marketing, 15% opacidade. Dentro do app: zero.

Iconografia

  • Base: Phosphor Icons (Regular weight, 1.5px stroke, 9k+ ícones, MIT)
  • Médicos específicos: Healthicons.org (gota com tipo sanguíneo, vacinas)

Border-radius (VARIA)

Cards 14px · Botões 10px · Inputs 10px · Modal/sheet topo 24px · Avatar 50% · Pills 999px · Chip pequeno 6px. A variação é o que cria hierarquia.

Acabamento (anti-flat)

  • Gradient base: jade-petróleo #0A1F1C → #134039 → #0F2A28 diagonal
  • Textura: noise/grain 2% opacity (evita flat AI-look)
  • Sombra dupla: próxima + distante tinted verde-petróleo
  • Marca d'água: monograma "VI" repetido diagonal 30°, 4% opacity
  • Holograma sutil: gradient cônico iridescente nas bordas
  • Aspect ratio: 1.585:1 (ISO/IEC 7810 ID-1, padrão internacional)

Animações

  1. Mount: scale 0.85→1.0 + translateY 20px→0, 500ms spring
  2. Parallax: desloca 0.5x do scroll geral
  3. Brilho ao tocar: shimmer 800ms
  4. Flip 3D: rotateY 180deg, 600ms — verso (histórico + emergência)
  5. Tilt giroscópio: long press, ±15° de inclinação
  6. Long-press: expande pra tela cheia (modo "mostrar pro médico")

07

RG público (tela que médico vê)

Aberto como web em vita.id/v/[token]. Médico não precisa instalar app.

Hierarquia clínica brutal:

  1. Header jade vita id + selo "✓ Identificação verificada"
  2. Foto + Nome 32px + Idade + A+ + "Doador: Sim"
  3. CARD VERMELHO DE ALERTA — border-left 4px, "⚠ ATENÇÃO MÉDICA CRÍTICA"
  4. Confirmação profissional (modal LGPD) — médico digita CRM antes de ver histórico
  5. Condições crônicas + CID-10
  6. Meds em uso contínuo
  7. Cirurgias recentes (últimos 12 meses)
  8. Contato emergência com botão "Ligar" verde (tap-to-call)

Audit trail invisível: IP, geo, timestamp, CRM digitado. Banco WORM. Push pro paciente: "Seu vita id foi acessado por Dr. X (CRM Y) às HH:MM em [cidade]."

08

Microinterações anti-cara-de-IA

As 12 regras absolutas:

  1. Sombras 2-camadas tinted (verde-petróleo, nunca preto puro)
  2. Easings cubic-bezier custom (nunca ease/ease-in-out padrão)
  3. 5 estados em todo componente (rest, hover, focus, pressed, disabled)
  4. Focus ring 2-4px com cor primary + offset branco
  5. Pressed: scale 0.97 + sombra reduzida em 80-120ms
  6. Stagger animation em listas (30-50ms entre itens)
  7. Skeleton que espelha conteúdo (não spinner genérico)
  8. Toasts de baixo no mobile, spring entrada
  9. Haptic seletivo (success/error/medium/light)
  10. Numbers count-up em métricas grandes
  11. Empty states com copy custom + CTA (nunca "ainda não há dados")
  12. Border-radius VARIA por componente (14/10/24/999 não tudo 12px)

Stack técnico

CSS puro + Web Animations API. Não Framer Motion (peso desnecessário pra PWA). Lottie só pra empty states ilustrados.

/* Easings vita id */
--ease-standard: cubic-bezier(0.25, 0.46, 0.45, 0.94);
--ease-spring:   cubic-bezier(0.34, 1.56, 0.64, 1);   /* botões, toggles */
--ease-apple:    cubic-bezier(0.32, 0.72, 0, 1);      /* modais */
--ease-exit:     cubic-bezier(0.4, 0, 1, 1);          /* saídas rápidas */

Checklist de aprovação (antes de cada tela)

20 itens. Se 5+ falham, refazer.

  • Tipografia tem 3+ tamanhos com contraste real
  • Hover state visível
  • Focus ring 2-4px com cor primary
  • Pressed state (scale 0.97 + sombra reduzida)
  • Disabled state (opacity 40% + cursor not-allowed)
  • Sombras 2-camadas tinted (não preto puro)
  • Border-radius VARIA entre componentes
  • Padding segue escala 4/8/12/16/24/32
  • Empty state tem copy custom + CTA
  • Loading é skeleton com forma do conteúdo
  • Nenhum gradiente verde→cyan ou roxo
  • Nenhum emoji como ícone de navegação
  • Modal abre com spring + backdrop blur
  • Listas têm stagger animation
  • Números grandes animam count-up
  • Easing cubic-bezier custom
  • Texto de erro específico
  • Nenhuma menção a "IA"
  • tabular-nums em TODO valor clínico
  • Toast vem de baixo no mobile com spring

3 Blue oceans (ninguém tem no Brasil)

  1. Selo de emergência via QR sem app — Apple Medical ID + RoadID no Brasil. Conecte SUS NÃO faz. Memed NÃO faz.
  2. Linha do tempo médica unificada — NINGUÉM no BR consolida consultas + exames + receitas + vacinas + alergias num feed cronológico filtrável.
  3. Portabilidade absoluta entre planos — brasileiro troca plano a cada 3-4 anos. vita id é o ÚNICO que segue com ele.
