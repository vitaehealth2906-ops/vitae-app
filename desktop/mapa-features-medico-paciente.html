Erros & Edge Cases
Tudo que pode dar errado — e como tratar para não frustrar médico nem paciente.
Retorno
Paciente sem conta vita id
CausaMédico tenta propor retorno mas paciente nunca criou RG.
O médico vêBotão "Marcar retorno" disponível, mas ao salvar: banner "Paciente não tem vita id — envie o link de cadastro".
SoluçãoCTA para copiar link de convite. Retorno fica como "pendente de cadastro do paciente".
Retorno
Data no passado
CausaMédico seleciona data que já passou no datepicker.
O médico vêValidação inline: "Selecione uma data futura". Datas passadas desabilitadas no calendário.
SoluçãoCalendário desabilita datas passadas. Campo nunca aceita data inválida.
Retorno
Retorno vencido sem comparecer
CausaData passou. Paciente não apareceu e não cancelou.
O médico vêBadge vermelho "Retorno vencido — 3 dias atrás". CTA: Reagendar ou Registrar como realizado.
O paciente vêCard âmbar "Você perdeu sua consulta de retorno" com botão para reagendar.
WhatsApp
Número inválido ou sem WhatsApp
CausaMédico digitou número com erro ou sem conta WhatsApp.
O médico vêValidação de formato no campo. Mas não conseguimos validar se tem conta WA (API paga). O link wa.me simplesmente não abre.
SoluçãoApós salvar, sugerir que médico teste o botão antes de ativar pro paciente. "Testar agora" manda pro próprio WhatsApp dele.
WhatsApp
Médico fecha config sem salvar
CausaMédico começa a configurar mas fecha a aba sem clicar "Salvar".
O médico vêSe tiver dados não salvos ao fechar: alerta "Você tem alterações não salvas. Deseja sair?"
SoluçãoAuto-save a cada mudança de campo (debounce 1s). Nunca perder configuração por acidente.
Documentos
Arquivo muito grande (>10 MB)
CausaMédico tenta subir PDF de 25 MB ou imagem de alta resolução.
O médico vêAntes do upload: "Arquivo muito grande. Máximo 10 MB. Comprimindo automaticamente..." (se imagem) ou erro se PDF.
SoluçãoImagens são comprimidas no browser antes do envio. PDFs: sugerir compressor online com link.
Documentos
Sem internet durante upload
CausaQueda de internet durante o upload de arquivo.
O médico vêBarra de progresso congela. Após 5s: toast "Sem conexão — tentando novamente...". Retry automático 3x.
SoluçãoIndexedDB guarda o arquivo localmente. Quando voltar a internet, resume do ponto onde parou.
Documentos
Mesmo arquivo enviado 2x
CausaMédico faz upload do mesmo PDF duas vezes por engano.
O médico vêSistema detecta pelo hash do arquivo. Aviso: "Este arquivo já foi enviado em 15/mai. Enviar novamente?"
SoluçãoDeduplicação por hash SHA-256. Médico decide se quer duplicar ou cancelar.
Geral
Paciente sem internet no celular
CausaPaciente abre o app sem internet para ver retorno ou documento.
O paciente vêCache local mostra última versão dos dados. Banner "Você está offline — dados podem estar desatualizados".
SoluçãoService Worker cacheia dados das consultas. Push notifications ficam enfileiradas e entregam quando reconectar.
